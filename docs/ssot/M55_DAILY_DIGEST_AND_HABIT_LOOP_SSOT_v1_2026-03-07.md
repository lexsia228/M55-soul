# M55 Daily Digest and Habit Loop SSOT
**Status:** SSOT (Law)  
**Version:** v1.0 (2026-03-07 JST)  
**Territory:** Web/App shared daily revisit loop, weekly light recap, DTR upsell timing, and digest messaging  
**Purpose:** Define the minimum repeatable loop that turns daily use into stored personal value and weekly paid curiosity without panic-selling.

---

## 0. Core Thesis
M55 should become a daily personal reflection habit.
The habit loop is:
1. small daily input
2. visible accumulation
3. calm weekly summary
4. deeper paid comparison
5. quiet return message

---

## 1. Canonical Loop Structure
### 1.1 Daily Habit Layer (Free)
Daily free layer includes:
- one short check-in
- current state update
- meter refresh
- continuity count
- quiet reason to return tomorrow

### 1.2 Weekly Light Layer (Free or lightly unlocked)
Weekly layer includes:
- recent pattern summary
- repeated themes
- directional changes
- summary of the last 7 days

### 1.3 Deep Paid Layer (DTR)
Deep paid layer includes:
- deeper interpretation of 7/30/90 day history
- repeated triggers
- comparison across time
- richer guidance framing
- higher-value personal archive use

---

## 2. Daily Check-in Rules
### 2.1 Input Budget
A daily check-in should take 30–90 seconds.

### 2.2 Required Daily Components
Minimum daily components:
- 1–3 selected state signals
- optional short note
- meter / zone update
- save confirmation

### 2.3 Daily Result
Daily result should feel calm and useful, not dramatic.

Allowed:
- "today's state" summary
- short reflective sentence
- continuity marker

Disallowed:
- prophecy
- certainty about other people
- scare copy

---

## 3. Weekly Light Summary Rules
### 3.1 Purpose
Weekly Light exists to convert saved data into visible value before paid conversion.

### 3.2 Content
May include:
- most frequent themes
- continuity count
- strongest directional shift
- one short interpretation block

### 3.3 Conversion Role
Weekly Light should naturally make the user curious about deeper comparison.
It must not hard-sell through fear or shame.

---

## 4. Deep DTR Role
### 4.1 Positioning
DTR is the premium deep-interpretation layer built on saved self-history.

### 4.2 Paid Trigger Logic
Strongest paid triggers are:
- enough history accumulated
- visible pattern emerging
- comparison window becomes meaningful
- user sees a question that the light layer cannot answer deeply

### 4.3 Not Allowed
DTR must not be sold with:
- panic claims
- manipulation of vulnerability
- false certainty

---

## 5. Daily Digest Email Rules
### 5.1 Purpose
Digest email exists to restart the user's reflection loop, not to act as a generic newsletter.

### 5.2 Allowed Content
Allowed:
- current zone/state summary
- number of days checked in this week
- recent theme labels in abstract form
- reminder of saved history value
- calm CTA back to hub

### 5.3 Disallowed Content
Disallowed:
- raw AI chat excerpts
- intimate memo bodies copied into email
- future prediction language
- manipulative scarcity
- guilt/shame pressure

### 5.4 Suggested Daily Digest Shape
Example structure:
- greeting / minimal identity-safe opener
- today's state
- this week's continuity
- one recent theme axis
- quiet CTA: continue reflection

---

## 6. Reward Logic
### 6.1 Free Reward Layer
M55 may provide a free lightweight weekly insight when continuity thresholds are reached.

Examples:
- short weekly reflection card
- simple pattern snapshot
- one-paragraph light DTR

### 6.2 Paid Reward Layer
Paid rewards remain:
- full DTR
- deeper comparisons
- monthly premium layer
- archive-informed interpretation

### 6.3 Canonical Behavioral Goal
The goal is not merely "daily opens."
The goal is:
- daily check-in habit
- weekly anticipation
- trust in stored history value
- repeat paid deep reads

---

## 7. Tone Rules
Digest and loop copy must feel:
- calm
- premium
- personal
- reflective
- non-panicked

Forbidden:
- "urgent relationship danger" style copy
- coercive fear
- mystical authority claims
- shame-based reactivation

---

## 8. Metrics (Product, not ad-tech)
Success metrics may include:
- daily check-in completion rate
- 7-day continuity rate
- weekly light view rate
- DTR conversion from weekly light
- digest return rate

Do not define success purely as email open rate or clickbait rate.
