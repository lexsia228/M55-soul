# DESKTOP PACK 精密スキャン報告（2026-02-26 JST）
対象: M55__DESKTOP_PACK_2026-02-01.zip
目的: Web版リリース（Stripe審査提出）に必要な「収益化ロジック／価格／財布分離／導線／法的器」要素の取りこぼし検査。

## 1. 結論（取りこぼし有無）
(1) 収益化ロジック（DTR・Boost・財布分離）に関して、Desktop Pack内に **Webへ転用すべき決定版の数値・ルールが存在** することを確認。
(2) 一方で、現在のWeb実装（DTR=¥1,000の単品）と **Desktop PackのDTR価格体系（Base/Complete）** に差分があるため、**「Webは暫定で¥1,000」「SSOTは将来Base/Completeへ合流」** のような扱いをSSOT上で明示しない限り、将来の合流時に矛盾が発生する。
(3) Stripe審査（+一般的なカードネットワーク審査）視点では、特商法だけでなく **プライバシーポリシー／利用規約／サポート導線** が常時到達可能であることが望ましい。Desktop Pack側でも「規約/プライバシーURLを揃える」「リンク切れは即死しやすい」旨の記述があり、Web側SSOTに取り込む必要がある。

## 2. Desktop Pack内で確認できた "決定版" 要素（Webに必須）
A. Boost（チャージ）価格（旧案が残っていてもB案で上書きする指示あり）
- 10回: ¥490
- 24回: ¥1,080
- 40回: ¥1,680
- ルール: まとめ買いほど1回単価が下がる（数値整合を全ファイルで保証するため、旧A案の記述が残る場合はB案へ統一）

B. 財布分離（Wallet Separation）
- Boostは「通常AIチャット／タロット」にのみ使用する共通財布。
- **DTR専用チャットにはBoostを使用しない（延長・再開不可）**。

C. DTR商品構造（Base / Complete）
- DTR Base: ¥2,400（税込） + DTR専用チャット無料枠 3往復（例）
- DTR Complete: ¥3,980（税込） + DTR専用チャット無料枠 8往復（例）
- DTR専用チャットは「買い切り＋無料枠＋利用期間」のセットで、無料枠消化 or 期限到来で入力不可。
- チャットが終わっても、DTR本文（静的DTR）は閲覧可能。

D. 規約/プライバシー/リンクの重要性（審査観点）
- 「利用規約 / プライバシーポリシー / サポート」を常に到達可能にする指示が複数箇所に存在。
- リンク切れは審査で致命傷になりうる旨の記述あり。

## 3. 現状Web実装との "衝突ポイント"（矛盾の芽）
1) DTR価格体系の差
- Web実装: DTR Core Static V1 を ¥1,000（想定）
- Desktop Pack: DTR Base/Complete を ¥2,400/¥3,980
=> **Webは暫定（プロトタイプ）で¥1,000を採用するなら、その旨をWeb SSOTに明記**し、将来のiOS SSOT（Base/Complete）へ合流する"移行方針"を記録して矛盾を封じる。

2) DTR専用チャットの無料枠
- Desktop Packは「例」として 3往復/8往復を提示。
- Web実装がDTR本文のみでDTR専用チャット未実装の場合、LP文言で「付属チャット」を断言しない。
=> **審査用LPは"現提供物"の事実だけ**にし、付属チャット導入時にSSOTを更新する。

3) 法的器の不足
- 特商法は実装済みでも、プライバシー/利用規約/サポート導線が未整備だと審査官の疑義が増える。
=> 最低3ページ（Privacy / Terms / Support）を "静的ページ" として追加し、共通フッターにリンクする。
