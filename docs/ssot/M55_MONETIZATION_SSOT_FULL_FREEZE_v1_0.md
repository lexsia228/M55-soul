## 0. 非交渉の原則（上位法）
1) 無料で成立：Freeは単体で体験として完結する。  
2) 支払いの意味：支払いは「深さ・保存・再訪性」＝時間・記憶・構造の価値。  
3) 購入価値は失われない：購入済み価値は失効・再ロック・再購入誘導をしない（恒久は恒久）。  
4) 途中課金禁止：儀式／鑑定／観測の途中で、結果や体験を"遮断して課金"させない。  
5) No Ads in Rituals：儀式・鑑定・解析中に広告UI/動画/演出を挿入しない。  
6) No Score / No Rank：ランキング、人気、スコア、％、達成演出は禁止。  
7) 言語の禁則：おすすめ/最適/成功/改善/当たる/診断/判定などの最適化・断定の語を避ける。  
8) 失敗は静寂：エラーや欠落は煽らず、嘘をつかず、静かな待機・固定文で扱う。  
9) 危機文脈では商用UIを止める：Crisis文脈（RED等）では商用導線を出さない（静かな固定文へ）。

## 1. 正規収益構造（v1.0の柱）
収益は以下4つのみ（Adsは中核にしない / 非主軸）。

A. サブスクリプション（iOS/Android公式IAP）
- Free：¥0
- Standard：¥680（保存30日・中層）
- Premium：¥1,980（保存90日・深層＋月次DTR）

B. DTR（Divination Trace Report）販売
- Static DTR：買い切り・恒久（例：本質/相性）
- Dynamic DTR：期間もの（例：週間/月間）
- Personal DTR：買い切り・恒久（サブスクと独立）

C. 保存・解析価値（Retention/Depth）
- 保存期間（7/30/90）と再訪性が価値。  
- 支払いは「結果の精度を変える」ためではなく「意味を積む場所」を開くため。

D. Chat Boost（補助）
- 共通消費型。中核ではない（広告同様、主軸にしない）。

## 2. プラン別の権利（回数・保存・閲覧）
※「見える/残る/戻れる」を差として設計する。結果を"良くする"は禁止。

### Free（¥0）
- Tarot：1回/日（簡易）
- AI Chat：1回/日（保存なし）
- 保存：当日分を中心（再訪性は最小）
- 広告：任意・補助的（体験条件にしない／儀式に入れない）

### Standard（¥680）
- 広告：非表示
- Tarot：5回/日
- AI Chat：5回/日
- 保存：30日
- Intent/観測の可視化：解放（ただしスコア化は禁止）

### Premium（¥1,980）
- 広告：非表示
- 各機能：実質無制限（ただし"無限"表現で煽らない）
- 保存：90日
- 月次DTR：付与（Premium特典）
- 深層解析・長期観測（ただし断定/最適化表現は禁止）

## 3. DTRの定義（本質と所有）
DTRの本質：鑑定結果ではなく「観測を意味構造として固定」したアーカイブ。

### 3.1 種類
1) Static DTR（恒久）
- 例：本質（CORE）、相性（SYNASTRY）
- 購入：買い切り
- 所有：恒久（失効なし・再購入なし・非表示化禁止）

2) Dynamic DTR（期限）
- 例：週間DTR／月間DTR（期間の"層"）
- 購入：買い切り（期間消費）または Premiumに包含
- 所有：期限付き（期限後は静かに眠る＝再表示しない／嘘にしない）

3) Personal DTR（恒久）
- 購入：全プラン購入可／サブスクと独立
- 所有：恒久（削除/失効/非表示禁止、再購入不要）
- アカウント紐づき（端末依存で失わない運用が前提）

### 3.2 価格（暫定プレースホルダー、後でAdmin変更可）
- Static DTR（本質/相性）：¥500〜¥1,200（買い切り・恒久）
- Dynamic DTR（週間/月間）：¥200〜¥500（期間消費）or Premium包含

## 4. 購入導線（Purchase Funnel）と購入後挙動（最重要）
### 4.1 購入導線の位置づけ
- DTRページがコアの収益導線。
- Homeは「入口＋ダイジェスト」であり、チェックアウト化しない。
- "買わせるための次行動"を強制しない（Dead End 0と両立）。

### 4.2 購入後の正規フロー（必須）
購入完了 → 対象ページへ遷移 → 上部「本質（CORE）」へ即ジャンプ（1回のみ）
- 演出なし（過剰な祝福/達成/ループ禁止）
- 以後は常に静止表示（更新/再生成/再購入誘導禁止）

### 4.3 再訪性（価値が失われないことの具体化）
- Home / MyPage / DTR一覧から常時アクセスできること。
- 既購入DTRは再販禁止。  
- MyPageは「アーカイブ入口」：Personal DTRは最上位に固定し、期限の影響を受けない。

## 5. 収益フック（Cross-Link Triggers）凍結
※煽り禁止。静かな提案としてのみ許可。儀式中に遮断課金をしない。

X-1 Tarot → DTR
- 条件：特定カード/キーワード出現
- UI：Result下部の静かなToast  
  文言（凍結例）：「この運命の『相性』を見る（DTR）」
- Tap：DTR購入LP → 購入 → 解析 → 相性ページTopへJump

X-2 AI Chat → DTR
- 条件：会話係数（例「不安」「将来」「相手」）が閾値超
- UI：AI提案  
  文言（凍結例）：「星の配置（本質DTR）を見てみますか？」
- Tap：DTR購入LPへ

X-3 AI Calendar → Dynamic DTR
- 条件：未来日付/特定吉日Tap
- UI：空予定を"埋める"のではなく、その期間の運勢（週間/月間DTR）の購入窓へ
- Tap：Dynamic DTR購入 → カレンダー上に"運勢が定着"（煽り無し、静止）

（補足）「カレンダー自体を課金ページにする」のは禁止。導線は最小で、押した結果が自然に"層として定着"する設計のみ許可。

## 6. タブ/ページ別の"収益の置き方"（導線マップ）
### 6.1 BottomNav（5タブ固定）
- Home：入口＋ダイジェスト（購入の主導線にしない）
- Tarot：日次の儀式。結果途中の課金遮断禁止。必要時のみX-1でDTR提案。
- AI Chat：会話。途中課金遮断禁止。必要時のみX-2でDTR提案。
- Premium鑑定：購買/深層の入口（ただしFOMO/煽り禁止、結果品質を変える説明禁止）
- MyPage：アーカイブ（Personal DTR最上位固定／Trace/Glossary/Export等入口）

※BottomNav第3タブは常にAI Chat（NOOP禁止）は別条文の上位ガード。

### 6.2 TopTabs（本質/相性/今日/週間/AIカレンダー）
- 本質（CORE）：Static DTR（買い切り・恒久）の到達点。購入後はここへジャンプ。
- 相性（Synastry）：Static DTR（相手ごとの買い切り・恒久）。結果途中の遮断課金禁止。
- 今日（Today）：Dynamic day。Freeは今日のみ、Std/Premは保存可（結果品質は不変）。
- 週間（Weekly）：Dynamic 7日。Free partial、Std full＋短期保存、Prem full＋長期保存。
- AIカレンダー：全員可視。保存/履歴はプラン準拠。過度な課金UIを置かない。

## 7. 広告ポリシー（非中核・厳格）
許可：Freeにおける補助的な任意報酬、体験の外側でのみ成立。  
禁止：
- 鑑定中・解析中・儀式中への挿入
- 広告視聴を体験条件にする
- 「広告再生 → 結果解放」構造
- 動画アフィリエイトを収益の柱として扱う
- 広告用の演出（BGM同期・UI連動）

---

## 8. 明示的禁止（監査で即FAIL）
- FOMOコピー、急がせる、今買わないと損、成功/改善/最適化の断言
- 途中課金遮断（結果や儀式を止めて課金）
- 購入済DTRの再販・再購入誘導
- ランキング/人気/数字/％/ゲージ/赤点/バッジ等の注意奪取UI
- Crisis文脈での商用導線表示
- 「支払うと結果が変わる」説明（人格/態度/占断の変化の示唆含む）

## 9. 実装に落とす最小チェック（収益化監査の観点）
- (1) すべての回数/保存/付与/クールタイムは policies/*.json のみを根拠にする（推測禁止）。
- (2) 購入後は「対象ページ遷移→COREへジャンプ（1回）」のみ。祝福演出なし。
- (3) DTR棚はStrata（地層）。順位/おすすめ/再販なし。
- (4) 儀式・鑑定・解析に広告を入れない。広告はFreeの外側・任意のみ。
- (5) 文言は"静かな提案"のみ（断言/最適化/煽り禁止）。
